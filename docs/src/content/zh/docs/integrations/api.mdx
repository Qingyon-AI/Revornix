# 三方 API

这篇文档说明 `/tp` 分组下的 API Key 接口。它们面向服务端集成、自动化流程和外部系统调用，能力范围覆盖文档采集、专栏处理、知识图谱和内容生产流程。

**本页所有接口都需要携带 `API-KEY` 请求头。**

| 请求头 | 值 |
| --- | --- |
| `API-KEY` | `your api key` |

API Key 可以在 Revornix 内通过 **左下角头像 -> 账户 -> APIKey 管理** 创建。

<Callout>
  如果你不想手写 HTTP 请求，优先参考 [Python SDK 与 CLI](./python-sdk) 或 [OpenClaw Skill](./openclaw-skill)。
</Callout>

## 通用约定

- 所有路径都以 `/tp` 开头。
- 大多数接口使用 `POST` 并接受 JSON 请求体。
- `/tp/file/upload` 使用 `multipart/form-data`。
- `/tp/document/create` 会读取可选请求头 `X-User-Timezone`，用于按用户时区计算当日 API 采集额度与月度汇总日期。
- `/tp/document/create` 后端会强制把 `from_plat` 记为 `api`，调用方不需要再额外传这个字段。

## 上传接口

### `POST /tp/file/upload`

先把文件上传到调用用户的默认文件系统，再通过 `/tp/document/create` 引用该路径创建文件文档。

#### 参数

| 参数名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `file` | File | 是 | 实际上传文件 |
| `file_path` | string | 是 | 写入默认文件系统的目标路径 |
| `content_type` | string | 是 | 文件 MIME 类型 |

#### 要求

- 调用用户必须已经配置默认文件系统。
- `file_path` 和 `content_type` 为必填字段。

#### 示例

```python
import httpx

client = httpx.Client()
headers = {
    "API-KEY": "your api key"
}
files = {
    "file": ("example.md", open("example.md", "rb"), "text/markdown")
}
data = {
    "file_path": "/files/example.md",
    "content_type": "text/markdown"
}

response = client.post(
    "http://localhost:8001/tp/file/upload",
    headers=headers,
    files=files,
    data=data,
)
```

上传完成后，通常下一步就是调用 `/tp/document/create`，把 `file_path` 作为文件文档的 `file_name` 传入。

### `POST /tp/section/create`

创建一个新的专栏。该接口会沿用应用内专栏的处理逻辑，包括触发策略、自动发布、自动播客、自动插图等配置。

#### 关键字段

| 参数名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `title` | string | 是 | 专栏标题 |
| `description` | string | 是 | 专栏描述 |
| `cover` | string | 否 | 专栏封面 |
| `labels` | int[] | 是 | 专栏标签 ID 列表 |
| `auto_publish` | boolean | 否 | 是否自动发布 |
| `auto_podcast` | boolean | 否 | 是否自动生成播客 |
| `auto_illustration` | boolean | 否 | 是否自动生成插图 |
| `process_task_trigger_type` | int | 是 | 处理触发方式，常见值为时间触发或更新事件触发 |
| `process_task_trigger_scheduler` | string | 否 | 定时触发时使用的 cron 表达式 |

#### 示例

```python
import httpx

client = httpx.Client()
headers = {
    "API-KEY": "your api key"
}

response = client.post(
    "http://localhost:8001/tp/section/create",
    headers=headers,
    json={
        "title": "AI 产品观察",
        "description": "按主题汇总行业动态与深度解读",
        "cover": "/covers/ai-weekly.png",
        "labels": [1, 2],
        "auto_publish": True,
        "auto_podcast": True,
        "auto_illustration": True,
        "process_task_trigger_type": 1
    },
)
```

### `POST /tp/document/create`

创建文档时，后端会按文档类型执行与应用内一致的采集与处理流程，并把 `from_plat` 固定记为 `api`。

#### 关键字段

| 参数名 | 类型 | 必填 | 描述 |
| --- | --- | --- | --- |
| `category` | int | 是 | 文档类型 |
| `sections` | int[] | 是 | 要推送到的专栏 ID 列表 |
| `labels` | int[] | 是 | 文档标签 ID 列表 |
| `title` | string | 否 | 文档标题 |
| `description` | string | 否 | 文档描述 |
| `cover` | string | 否 | 文档封面 |
| `content` | string | 条件必填 | 速记文档时必填 |
| `url` | string | 条件必填 | 链接文档时必填 |
| `file_name` | string | 条件必填 | 文件文档时必填，对应上传时的 `file_path` |
| `auto_summary` | boolean | 否 | 是否自动生成摘要 |
| `auto_podcast` | boolean | 否 | 是否自动生成播客 |
| `auto_tag` | boolean | 否 | 是否自动打标 |

#### 分类

| `category` | 含义 |
| --- | --- |
| `0` | 文件文档 |
| `1` | 链接文档 |
| `2` | 速记文档 |
| `3` | 音频文档 |

#### 要求

- 调用方不需要传 `from_plat`，服务会自动写成 `api`。
- 官方部署下，这个接口会按套餐能力校验 API 采集额度。
- 官方部署下，网站文档和文件文档在累计数量较高时也会触发额外能力校验。
- 如果你依赖“按用户当天”统计额度，建议显式传 `X-User-Timezone`。

#### 示例

```python
import httpx

client = httpx.Client()
headers = {
    "API-KEY": "your api key",
    "X-User-Timezone": "Asia/Shanghai",
}

response = client.post(
    "http://localhost:8001/tp/document/create",
    headers=headers,
    json={
        "category": 1,
        "url": "https://example.com/article",
        "sections": [12],
        "labels": [3],
        "auto_summary": True,
        "auto_podcast": False,
        "auto_tag": True
    },
)
```

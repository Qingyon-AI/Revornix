# 管理台

管理台提供一套独立于普通工作台的后台界面，入口位于 `/admin`。它用于面向平台管理者执行高频的资源巡检、用户维护和内容清理操作。

## 访问权限

<Callout>
	只有 `ADMIN` 和 `ROOT` 用户可以进入管理台。页面入口和路由都会做权限校验，普通用户无法直接访问。
</Callout>

管理台使用独立的 Next.js 路由分组与布局，和普通用户侧的工作台页面分开维护。

## 页面结构

当前管理台包含以下模块：

- 总览页：显示后台工作区入口，并集中跳转到用户、文档和专栏管理
- 用户管理：浏览、筛选、查看、创建、编辑、删除用户
- 文档管理：浏览、搜索、查看、删除文档，并跳转到实际文档详情页
- 专栏管理：浏览、搜索、查看、删除专栏，并跳转到实际专栏详情页
- 用户通知管理：按用户维度管理通知任务

## 用户管理

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260408171507073.png)

用户管理页主要用于处理账号和权限相关操作，当前支持：

- 按昵称、邮箱等关键词搜索用户
- 按角色、状态筛选用户
- 分页浏览用户列表，并切换每页条数 `10 / 20 / 50`
- 新增用户
- 编辑用户基础资料，包括昵称、邮箱、角色、状态、个性签名和头像
- 删除用户
- 查看用户详情
- 跳转到用户真实详情页 `/user/detail/[id]`
- 查看当前可用额度和点数账本
- 进入该用户的通知任务管理页

其中头像不是手动填写路径，而是直接上传实际图片并预览。

所有创建、更新、删除类操作在真正提交前都会弹出确认框，避免误操作。

## 文档管理

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260408171608478.png)

文档管理页用于处理平台侧文档内容，当前支持：

- 按标题或描述搜索文档
- 分页浏览文档列表，并切换每页条数 `10 / 20 / 50`
- 查看文档基础信息
- 删除文档
- 一键跳转到实际文档详情页 `/document/detail/[id]`

为了适配后台场景，超长标题会在列表中自动截断，避免把布局撑坏。

## 专栏管理

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260408171604653.png)

专栏管理页当前支持：

- 按标题或描述搜索专栏
- 分页浏览专栏列表，并切换每页条数 `10 / 20 / 50`
- 查看专栏基础信息
- 删除专栏
- 一键跳转到实际专栏详情页 `/section/detail/[id]`

## 用户通知管理

管理员可以从用户管理页进入某个用户的通知管理页，路由为 `/admin/users/[id]/notifications`。

这一页主要用于直接维护该用户的通知任务，当前支持：

- 浏览该用户的通知任务列表
- 分页查看通知任务
- 新增通知任务
- 编辑通知任务
- 删除通知任务
- 启用或停用通知任务
- 在表单内选择该用户当前可用的通知源与通知目标
- 为通知内容上传实际封面图片，而不是手动填写图片路径

通知任务表单采用顶部固定、底部固定、中间滚动的布局，适合处理字段较多的后台配置场景。

## 交互约定

管理台当前遵循以下交互规则：

- 列表页普遍支持刷新、搜索、分页和每页条数切换
- 删除、创建、更新等写操作都要求二次确认
- 用户、文档、专栏都可以从后台一键跳到真实详情页
- 管理页会优先使用表格和明确操作列，便于高密度后台操作

## 与普通设置页的关系

普通用户仍然通过设置页管理自己的通知源、通知目标和通知任务。管理台中的通知能力，主要用于管理员代替目标用户查看和调整通知任务配置。

如果你需要了解普通用户侧的通知配置方法，请继续阅读 [通知管理](./notification)。

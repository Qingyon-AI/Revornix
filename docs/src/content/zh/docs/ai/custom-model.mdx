# 自定义模型

Revornix 的“模型”不是单一聊天入口配置，而是会被不同工作流分别消费。  
现在最重要的两个默认模型位是：

- `Revornix AI 默认模型`
- `文档阅读 / 总结默认模型`

这两个模型的职责已经明确分开。

## 1. 模型入口的作用

### Revornix AI 默认模型

这个模型主要用于 Revornix AI 对话本身。  
当你在产品里和 Revornix AI 交互时，默认会优先走这里配置的模型。

### 文档阅读 / 总结默认模型

这个模型不仅用于文档总结。  
在这些工作流里，它还会参与：

- 文档总结
- 标题与描述生成
- 专栏内容处理
- 知识图谱相关抽取
- 某些引擎链路中的辅助文本理解

<Callout>
	所以“文档阅读 / 总结模型”现在更接近一个通用内容理解模型，而不是只负责生成一段 Summary。
</Callout>

## 2. 支持的模型协议

自定义模型以兼容 OpenAI 协议为主。  
也就是说，第三方模型只要能够通过兼容 OpenAI API 的方式接入，就可以在 Revornix 中作为可选模型使用。

## 3. 模型配置流程

模型配置大致分成三层：

1. 创建模型供应商
2. 在供应商下增加模型
3. 把某个模型设为默认用途

其中：

- 供应商名称和描述主要用于你自己区分来源
- 模型名称必须填写真实模型名，不能随意自定义
- 模型设置完成后，还需要到“默认模型选择”里把它绑定到具体用途

## 4. 套餐与可访问性规则

现在模型不只是“能否看见”的问题，还带有访问等级限制。

模型资源支持以下访问控制：

- 模型可以设置 `required_plan_level`
- 默认模型在写入用户设置前会做一次访问校验
- 如果账号套餐不满足，模型可以出现在列表里，但会被禁用或提示不可用

这意味着：

- 你可以配置模型社区中的公开模型
- 但默认模型选择和实际使用仍会受当前套餐限制影响

## 5. 官方托管模型

模型支持“官方托管模型”这一类标记。

它的几个重要含义是：

- 模型可被标记为 `is_official_hosted`
- 官方托管模型可以配置积分倍率 `compute_point_multiplier`
- 在官方部署环境中，系统会 seed 一套 Revornix 官方模型供应商

官方初始化会创建以下模型供应商：

- `Revornix`

官方初始化会创建以下默认模型项：

- `gpt-5.4`

这里列出的是官方初始化中提供的默认模型项。  
具体请求路由会根据平台配置和资源策略决定。

## 6. 模型社区与 Fork

模型社区能力仍然存在：

- 你可以把自己的模型供应商设为公开
- 其他用户可以在社区里看到公开模型供应商
- 如果想真正使用对方的模型供应商，需要先 Fork

Fork 的意义是把这份资源纳入你自己的可用资源集合，否则默认模型选择器不会把它作为你自己的候选资源。

## 7. 和引擎的边界

当前产品里，“模型”和“引擎”不要混为一谈：

- 模型：更偏向 LLM / 推理 / 阅读 / 对话能力
- 引擎：更偏向网站解析、文件解析、播客合成、音频转写、图片生成这类专门能力

你在大多数功能里，实际需要同时配置这两类默认资源，而不是只配模型或只配引擎。

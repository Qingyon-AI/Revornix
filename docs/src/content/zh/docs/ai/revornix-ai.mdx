# Revornix AI

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260417163541796.png)

Revornix AI 不是一个孤立聊天框，而是建立在以下几层能力之上：

- 默认 Revornix AI 模型
- 文档 / 专栏内容上下文
- 向量检索
- 知识图谱扩展
- MCP Client 与 MCP Server

## 1. 默认模型位

要正常使用 Revornix AI，至少需要配置：

- `默认 Revornix AI 模型`

而在很多更深的工作流里，还会间接依赖：

- `默认文档阅读 / 总结模型`

因为 Revornix AI 不只是普通聊天，它会和文档、专栏、图谱、播客等结果共享同一套知识生产链路。

## 2. 上下文来源

Revornix AI 的上下文并不只来自一段静态系统提示。

在面向文档和专栏的场景中，它会结合：

- 文档或专栏的 Markdown 内容
- 相关文档片段召回
- 向量检索结果
- 图谱扩展结果
- MCP 能力返回结果

这也是为什么 Revornix AI 能更像“对你的知识库做操作”，而不是只做一个外部聊天窗口。

## 3. MCP 仍然是核心组成部分

客户端支持接入：

- HTTP MCP
- Std MCP

服务端侧能力内置在 API 里，主要分布在：

- `api/mcp_router/common`
- `api/mcp_router/document`
- `api/mcp_router/graph`

因此 Revornix AI 现在既可以主动调用文档 / 图谱能力，也可以被外部 MCP Client 反向调用。

## 4. 官方托管与自配资源并存

Revornix AI 并不要求你只能使用一种资源来源。

你可以：

- 自己配置模型供应商和模型
- 使用社区公开并 Fork 到自己名下的模型
- 在官方部署环境中使用官方托管模型

官方初始化会创建：

- 官方模型供应商 `Revornix`
- 默认官方模型项 `gpt-5.4`

但具体官方路由能力并不等于“所有请求永远绑定到某个单一模型名”。

## 5. 套餐与访问限制

Revornix AI 的默认模型选择、实际调用和部分官方托管资源，都会受套餐等级影响。

所以现在更准确的理解方式是：

- Revornix AI 是一个建立在“默认模型 + 默认资源 + 知识上下文 + MCP”之上的综合能力
- 它的可用性并不只取决于有没有配置 API Key，还取决于资源是否被套餐允许访问

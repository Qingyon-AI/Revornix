# 网站快照

网站快照用于保留网站文档在不同时间点的解析结果，方便回看同一链接在过去某个时刻的内容状态。

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260406181338175.png)

## 1. 生成方式

网站快照不是单独手动创建的功能，它和网站文档的 Markdown 转换流程绑定在一起：

1. 创建网站文档
2. 网站内容被解析并转换为 Markdown
3. 转换成功后，系统自动保存一份网站快照

如果同一篇网站文档后续再次触发 Markdown 转换并成功，系统会继续追加新的快照记录。

## 2. 快照里包含什么

每条网站快照会保存一组和当次解析结果对应的数据：

- 原始链接
- 当时解析出的标题
- 当时解析出的描述
- 当时解析出的封面
- 对应的 Markdown 文件
- 快照生成时间

这意味着你看到的不是“当前网站重新抓取后的结果”，而是当次转换完成时保留下来的历史内容版本。

## 3. 在哪里查看

网站快照显示在**网站文档详情页**里。

当某个网站文档存在多次转换记录时，详情页会出现快照切换器，你可以：

- 查看这个链接累计保存了多少份快照
- 按时间选择某一份历史快照
- 阅读该时间点对应的 Markdown 内容

选中快照后，正文区域会切换到那一版历史内容，而不是始终显示最新解析结果。

## 4. 和重新转换的关系

网站快照和“重新转换 Markdown”是联动的：

- 当文档还没有成功转换时，你可以在详情页手动触发转换
- 当你主动重试转换并成功后，会新增一份快照
- 快照数量会随着成功转换次数增加

所以网站快照更适合被理解成“网站文档的版本历史”，而不是独立的收藏夹或书签能力。

## 5. 适合的场景

网站快照尤其适合这些场景：

- 跟踪同一网页在不同时间的内容变化
- 回看某次解析时实际进入知识库的 Markdown 结果
- 对照标题、摘要、封面等元数据在不同时间点的变化

如果你还把网站文档推送进了专栏，那么这些历史快照也能帮助你理解后续总结、图谱或播客结果是基于哪一版内容生成的。

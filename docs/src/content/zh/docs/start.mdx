# 快速开始

<Callout>
	如果你希望先直接体验产品，请访问 <a href='https://app.revornix.com'>https://app.revornix.com</a>。
	本页主要用于本地/自部署场景。
</Callout>

## 手动部署方式

<Callout>
	强烈建议使用 conda 针对每个服务创建不同的 python 虚拟环境，因为不同服务之间的
	python
	依赖可能存在冲突。当然如果你有别的python虚拟环境管理工具，也可以使用别的。
</Callout>

<Steps>

### 克隆仓库到本地

```shell
git clone git@github.com:Qingyon-AI/Revornix.git
cd Revornix
```

### 安装并且启动基础服务

<Callout>
	如果你本地还没有 Postgres、Redis、Neo4j、MinIO、Milvus，可以直接使用 `docker-compose-local.yaml` 和 `.env.local.example` 启动本地基础依赖层。
</Callout>

<Callout type='warning'>
	注意：如果你本地已经有部分依赖在运行，请按实际情况关闭 `docker-compose-local.yaml` 中对应服务，避免端口冲突或数据串用。
</Callout>

复制样例文件，并结合[Revornix 环境变量章节](environment)按实际需要修改。对于最简单的本地场景，通常最关键的是先设置好 `OAUTH_SECRET_KEY`。

```shell
cp .env.local.example .env.local
```

启动本地基础设施服务。

```shell
docker compose -f ./docker-compose-local.yaml --env-file .env.local up -d
```

### 各大微服务环境变量配置

```shell
cp ./web/.env.example ./web/.env
cp ./api/.env.example ./api/.env
cp ./celery-worker/.env.example ./celery-worker/.env
```

`hot-news` 当前仓库里没有提交 `.env.example`，如果你需要自定义它的配置，可以参考[Revornix 环境变量章节](environment)里的变量说明，手动创建 `hot-news/.env`。

然后分别进入对应服务的环境变量文件进行配置，详情见[Revornix环境变量章节](environment)。

<Callout type='warning'>
	注意：如果你使用手动部署，`OAUTH_SECRET_KEY` 必须在 API 和 Worker 之间保持一致，否则认证和内部调用信任链会失效。
</Callout>

### 初始化一些必要的数据

```shell
cd api
python -m data.milvus.create
python -m data.sql.create
```

### 启动核心后端服务

```shell
cd api
conda create -n api python=3.11 -y
conda activate api
pip install -r ./requirements.txt
fastapi run --port 8001
```

### 启动热搜聚集服务

```shell
cd hot-news
pnpm install
pnpm build
pnpm start
```

### 启动 celery 任务序列

```shell
cd celery-worker
conda create -n celery-worker python=3.11 -y
conda activate celery-worker
pip install -r ./requirements.txt
playwright install
celery -A common.celery.app worker --pool=threads --concurrency=20 --loglevel=info -E
```

### 启动前端服务

```shell
cd web
pnpm install
pnpm build
pnpm start
```

</Steps>

当你将所有服务都启动之后，访问 http://localhost:3000 即可看到前端页面。

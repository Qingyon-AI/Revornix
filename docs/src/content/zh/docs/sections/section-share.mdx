# 专栏分享与共创

专栏分享与共创能力支持把专栏内容分享给指定用户，或发布到公开网络，适合团队协作和公开传播两类场景。

## 分享方式

专栏分享主要包括以下方式：

1. 邀请指定用户并且授予权限。

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260417163242591.png)

你可以通过专栏底部的分享按钮，搜索同平台内的用户，选择共享对象并授予权限。系统提供三种权限：

- **完整权限（FULL_ACCESS）**：在读写能力之外，额外具备邀请新成员、审批入栏申请，以及管理由自己邀请/审批进来的成员的能力。
- **读写权限（READ_AND_WRITE）**：可以查看专栏，编辑专栏元数据（标题、描述、封面、标签、自动播客/插图开关），编辑专栏内文档的 Markdown 正文，将文档加入专栏，对处理失败的文档发起重试。
- **只读（READ_ONLY）**：可以查看专栏、记笔记、收藏、评论、使用 AI 问答；不能编辑专栏或其内容。

### 身份与能力矩阵

身份只表达关系（创建者 / 协作者 / 订阅者），权限只表达协作能力（FULL_ACCESS / READ_AND_WRITE / READ_ONLY）。具体能力按下表分配：

| 操作 | 创建者 | FULL_ACCESS | READ_AND_WRITE | READ_ONLY |
| --- | :---: | :---: | :---: | :---: |
| 阅读、评论、笔记、收藏、AI 问答 | ✓ | ✓ | ✓ | ✓ |
| 编辑专栏元数据（标题/描述/封面/标签/自动开关） | ✓ | ✓ | ✓ | ✗ |
| 将文档加入专栏、重试专栏内文档处理 | ✓ | ✓ | ✓ | ✗ |
| 邀请新成员 / 审批入栏申请 | ✓ | ✓ | ✗ | ✗ |
| 修改、移除成员 | 所有成员 | 仅自己邀请/审批进来的成员 | ✗ | ✗ |
| 触发专栏整体处理 / 生成 PPT / 生成播客 | ✓ | ✗ | ✗ | ✗ |
| 发布 / 取消发布 | ✓ | ✗ | ✗ | ✗ |
| 删除专栏 | ✓ | ✗ | ✗ | ✗ |

<Callout type='info'>FULL_ACCESS 不是"小一号的创建者"。它能把人请进来，但只能管理自己请进来的人；发布、删除、整体处理这类影响所有读者或整张专栏的操作仍然只属于创建者。这条规则同时由后端 API 强制和前端 UI 隐藏，绕过 UI 直接调接口也会被拦住。</Callout>

2. **发布网络**：你可以将专栏发布到公开网络。发布后，其他用户可以在社区专栏列表中搜索并查看该专栏内容，公开链接也可以用于外部传播。

## 社区与公开 SEO 页面

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260522172606158.png)

公开发布后的专栏会进入社区公开入口，主要包含多层页面：

- 社区页：`/community`，用于浏览和搜索已发布专栏与文档
- 公开专栏页：`/section/[publish_uuid]`，用于展示某个已发布专栏的公开详情
- 公开文档页：`/document/[id]`，用于展示某个已发布文档的公开详情
- 公开用户页：`/user/[id]`，用于展示创作者公开资料与其已发布专栏
- 公开热搜页：`/hot-search`，用于在不进入私有工作区的情况下浏览标准化热榜卡片

社区页会展示：

- 公开专栏与公开文档两个视图
- 按当前视图展示的公开标签筛选
- 当前结果集数量与上下文信息
- 关键词搜索入口
- 公开专栏或公开文档卡片

公开专栏页会展示：

- 专栏标题、描述、封面、标签、创建者与更新时间
- 文档数量、订阅人数、评论数量等公开指标
- 专栏 Markdown 正文
- 已公开的播客信息
- 专栏图谱展示
- 关联文档列表
- 评论区与订阅入口

公开专栏页使用发布时生成的 `publish_uuid` 作为对外地址标识。只要发布地址有效，外部访问者就可以直接查看对应专栏内容。

## 搜索与发现

社区页支持按关键词、公开标签筛选公开专栏与公开文档，并按分页继续浏览更多结果。公开专栏卡片中会同时展示：

- 专栏封面与标题
- 简介与标签
- 创建者头像与昵称
- 文档数与订阅数
- 订阅按钮

文档结果会展示标题、简介、标签、创建者和相关公开导航。点击创建者头像或昵称后，可以继续进入对应的公开用户页，查看该用户的公开资料以及更多已发布专栏或文档。

## 订阅机制

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260522172831169.png)

专栏发布后，其他用户可以查看内容，并通过专栏底部操作栏中的订阅按钮完成订阅。

订阅后，可以在系统左侧菜单栏中的订阅专栏列表里查看该专栏，持续跟踪更新。

如果你还配置了[通知任务](../workspace/notification)，也可以结合 `SectionProcessCompleted`、`SectionContentUpdated`、`SectionSubscribed` 等事件，把专栏处理完成、内容编辑和订阅相关动态发送到邮件、飞书、钉钉、Telegram 或 Apple 设备。

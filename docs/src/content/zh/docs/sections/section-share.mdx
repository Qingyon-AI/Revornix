# 专栏分享与共创

专栏分享与共创能力支持把专栏内容分享给指定用户，或发布到公开网络，适合团队协作和公开传播两类场景。

## 分享方式

专栏分享主要包括以下方式：

1. 邀请指定用户并且授予权限。

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260417163242591.png)

你可以通过专栏底部的分享按钮，搜索同平台内的用户，选择共享对象并授予权限。系统提供三种权限：

- **完整权限**：用户可以查看、编辑和修改专栏内容，同时也具备将专栏进行二次分享的能力（即可以继续邀请他人共建）。
- **读写权限**：用户可以查看、编辑和修改专栏内容。
- **只读**：用户可以查看专栏内容，但不能进行编辑或修改。

<Callout type='warning'>目前仅有专栏的创建者具备删除其他共享用户以及修改其他共享用户权限的能力。非创建者一旦邀请完成其他用户，则不具备修改对应被邀请用户权限以及删除对应被邀请用户共享状态的能力。</Callout>

2. **发布网络**：你可以将专栏发布到公开网络。发布后，其他用户可以在社区专栏列表中搜索并查看该专栏内容，公开链接也可以用于外部传播。

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260417163158656.png)

Revornix 提供了刷新发布地址的机制。点击专栏发布窗口中的刷新按钮后，会生成新的访问 URL。访问者视角如下：

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260417163331282.png)

## 社区与公开 SEO 页面

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260417163403320.png)

公开发布后的专栏会进入社区公开入口，主要包含三层页面：

- 社区页：`/community`，用于浏览和搜索已发布专栏
- 公开专栏页：`/section/[publish_uuid]`，用于展示某个已发布专栏的公开详情
- 公开用户页：`/user/[id]`，用于展示创作者公开资料与其已发布专栏

社区页会展示：

- 已发布专栏总数
- 当前列表中的文档总量与订阅量聚合
- 关键词搜索入口
- 专栏卡片列表

公开专栏页会展示：

- 专栏标题、描述、封面、标签、创建者与更新时间
- 文档数量、订阅人数、评论数量等公开指标
- 专栏 Markdown 正文
- 已公开的播客信息
- 专栏图谱展示
- 关联文档列表
- 评论区与订阅入口

公开专栏页使用发布时生成的 `publish_uuid` 作为对外地址标识。只要发布地址有效，外部访问者就可以直接查看对应专栏内容。

## 搜索与发现

社区页支持按关键词搜索公开专栏，并按分页继续浏览更多结果。公开专栏卡片中会同时展示：

- 专栏封面与标题
- 简介与标签
- 创建者头像与昵称
- 文档数与订阅数
- 订阅按钮

点击创建者头像或昵称后，可以继续进入对应的公开用户页，查看该用户的公开资料与更多已发布专栏。

## 订阅机制

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260318192803384.png)

专栏发布后，其他用户可以查看内容，并通过专栏底部操作栏中的订阅按钮完成订阅。

订阅后，可以在系统左侧菜单栏中的订阅专栏列表里查看该专栏，持续跟踪更新。

如果你还配置了[通知任务](../workspace/notification)，也可以结合 `SectionUpdated`、`SectionSubscribed` 等事件，把专栏更新和订阅相关动态发送到邮件、飞书、钉钉、Telegram 或 Apple 设备。

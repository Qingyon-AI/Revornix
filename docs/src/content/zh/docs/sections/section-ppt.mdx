# 专栏 PPT

Revornix 支持基于专栏内容生成 PPT 预览结果。这是专栏详情页中的正式工作流之一，而不是外部插件能力。

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260406180744647.png)

## 1. 入口位置

PPT 能力位于专栏详情页的媒体区域，和专栏播客并列展示。

当专栏满足生成条件时，你可以直接：

- 提交 PPT 生成任务
- 查看生成中的幻灯片预览
- 在生成完成后轮播浏览每一页
- 下载最终的 `.pptx` 文件（如果后台已生成）

## 2. 生成依赖

PPT 生成依赖以下条件：

- 专栏已经存在可用的 Markdown 内容
- 默认文档阅读/总结模型可用
- 默认图片生成引擎可用

如果这些资源没有准备好，界面会直接给出设置入口，而不是只报一个模糊错误。

## 3. 预览形态

生成中的时候：

- 页面会展示任务处于等待或处理中
- 如果已有部分幻灯片完成，会优先显示已出图的页

生成成功后：

- 会显示 PPT 标题、副标题等预览信息
- 支持上一页 / 下一页切换
- 幻灯片预览图支持点击放大查看
- 如果已有 `.pptx` 文件，会提供下载按钮

## 4. 这项能力和专栏工作流的关系

PPT 不是孤立生成的，它依赖专栏已有的内容结果，所以当专栏文档集合变化后，PPT 也可能需要重新生成。

从工作流角度看，它通常位于这些结果之后：

1. 文档进入专栏
2. 专栏处理生成 Markdown
3. 专栏内容稳定后生成播客 / PPT / 图谱等衍生产物

因此更准确的理解方式是：PPT 是“专栏内容产物”的一种，而不是单独上传模板后生成的外部文件功能。

# 专栏

专栏页是围绕主题内容展开的工作台，而不只是展示一段总结文本。  
一个专栏通常会同时包含：

- 专栏 Markdown 正文
- 关联文档列表
- 专栏知识图谱
- 专栏播客
- 专栏 PPT 预览与下载结果
- 分享、订阅、评论、协作等交互能力

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/20260318191950847.png)

## 1. 文档绑定仍然是专栏的上游

专栏和文档依然是高强度绑定关系，但现在的影响范围更大了。  
文档被加入专栏后，不只是“出现在列表里”，还可能触发：

- 专栏正文重生成
- 专栏播客更新
- 专栏 PPT 更新
- 专栏知识图谱结果变化

把文档加入专栏主要有两种方式：

- 创建文档时直接选中专栏
- 在专栏详情页的文档面板中继续添加文档

<Callout>
	如果你只是订阅者或只读成员，就不能修改专栏内容，也不能触发这些更新链路。
</Callout>

## 2. 专栏详情页的主要区域

### Markdown 正文

专栏正文来自后台生成的 Markdown 文件。  
如果专栏正在处理中，正文区会显示处理中骨架；如果是定时触发型专栏且尚未到触发时间，会显示“等待下一次调度”的提示。

### 文档列表

你可以在专栏详情页查看专栏包含的全部文档，并继续维护这个集合。

### 专栏内 AI 问答

专栏支持范围内 AI 问答。  
这类问答不只是读专栏最终总结，还会结合专栏 Markdown、相关文档片段、向量召回和图谱扩展结果来组织上下文。

### 评论、分享、订阅、协作

专栏详情页保留了评论、分享、订阅和成员协作相关操作。  
这些能力共同构成了专栏从个人知识整理到公共内容协作的完整链路。

## 3. 专栏播客

专栏可以独立生成播客：

- 如果开启了自动播客，专栏内容更新后会继续进入播客链路
- 如果没有开启，也可以在详情页手动生成
- 如果内容已经变化，界面会提示播客结果可能已过期，并允许重新生成

成功后，详情页会直接提供播放器，而不是跳到外部页面。

## 4. 专栏 PPT

专栏支持基于内容生成 PPT 预览结果，具体表现为：

- 可以手动提交 PPT 生成任务
- 生成中会显示当前已完成的幻灯片预览
- 生成成功后可以在专栏详情页轮播查看每一页
- 每张幻灯片预览图都支持点击放大
- 如果后台已经生成 `.pptx` 文件，界面会直接提供下载入口

这项能力依赖：

- 默认文档阅读/总结模型可用
- 默认生图引擎可用
- 专栏已经有可用的 Markdown 内容

如果这些条件不满足，界面会直接提示你去设置相关资源，而不是静默失败。

## 5. 专栏知识图谱

每个专栏仍然拥有自己的知识图谱。  
不过现在它更适合被理解成“专栏聚合结果的一部分”，因为它会随着专栏文档集合变化而一起刷新。

## 6. 专栏浏览维度

专栏列表支持：

- 我的专栏
- 社区专栏
- 已订阅专栏
- 标签筛选
- 标题搜索
- 时间排序

所以专栏既可以是个人工作流容器，也可以是公共内容载体。

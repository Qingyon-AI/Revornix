# Revornix-v0.9.1

距离 v0.9.0 已积累 40+ 次提交。本次发布聚焦在账户安全、编辑器能力、内容存储统一、通知体系与处理流程的稳健性，以及一批底层可观测性与 UI 细节的打磨。

## 更新亮点

### 账户安全与登录

- **新增 MFA（多因素认证）**：账户可启用 TOTP 与 Passkey，登录与敏感操作多一层保护。
- **登录方式删除保护**：当某个登录方式是账户唯一凭证时，禁止误删，避免把自己锁在门外。
- **微信登录与账号绑定流程重做**：绑定、解绑与登录路径更清晰一致。
- **OAuth 登录优化与修复**：修复了若干登录异常，整体登录体验更稳。

### Markdown 编辑器

- 新增 **地图节点（Map）**：可在文档中插入并编辑地图，并对其交互做了进一步优化。
- 新增 **更多自定义节点** 与 **Slash 命令**：输入 `/` 即可快速插入各类块，写作更顺手。
- 修复了编辑器在多种场景下的渲染与交互问题。

### 内容存储统一

- **快速笔记内容迁移到 Markdown 文件**：统一了 quick-note 的存储与保存逻辑，并修复了相关 Markdown 展示问题。
- **音频笔记内容迁移到 Markdown 文件**：音频转写结果以 md 文件统一管理，便于检索与复用。

### 音频文档

- 新增 **会议模式（Meeting Mode）**：针对会议录音优化转写，支持说话人分离场景。
- 重做 **音频播放器** 交互与展示。

### 通知体系

- **处理完成通知时机重构**：文档 / 专栏「处理完成」通知改由编排层在**所有后续任务（向量化、构图、摘要、播客）真正结束后**统一发出，不再「子任务还在跑就提示完成」，也不再重复发送。
- **接收人更准确**：处理完成、内容编辑、评论回复等通知补齐了应收到却被遗漏的角色（如专栏创建者、被回复评论的作者）。
- **通知发射与消费分层**：内层处理工作流不再耦合通知逻辑，发射侧（`notification/emit`）与消费侧（`notification/dispatch`）职责清晰。
- 精简了不必要的「局部小任务完成」通知，保留整体流程完成提醒。

### 知识图谱

- 知识图谱支持在「知识节点关联」与「文档节点关联」两种视角间切换，并加入图例、统一工具栏等交互优化。

### 文件系统

- 文件系统进一步优化：删除文档 / 专栏时同步清理对应的远程文件资源，避免存储泄漏。

### 社区与页面

- 新增 **公开文档页面**，便于浏览与分享公开内容。
- 优化 **热搜页面** 与 dashboard 的加载体验。

### 可观测性与底层

- **日志与链路追踪迁移**：统一了 `web` / `api` / `celery-worker` 的日志与 traces 采集（参见 `OBSERVABILITY.md`）。
- 为 worker 增加了更细的运行日志；为 Milvus 连接增加超时保护。
- 优化前端异步加载；将 `instrumentation.ts` 迁移到 `src/`；修复服务端请求的 headers 透传。

### UI 细节与快捷键

- 重做聊天框输入区：左下角改为 `+` 菜单收纳上传等操作，文档 / 专栏 / Revornix AI 三处交互统一。
- 新增若干快捷键。
- 一批界面细节优化与 404 / 跳转问题修复。

### Bug 修复

- 修复管理后台的若干控制问题。
- 修复 OAuth 登录、快速笔记 Markdown、编辑器等处的细节缺陷。
- 清理了一批无用代码。

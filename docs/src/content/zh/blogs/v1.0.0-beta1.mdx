# Revornix-v1.0.0-beta1

这是迈向 1.0 的第一个公测版本。本次发布围绕两条主线：把「公开分享」做完整（访问密钥体系），以及在 1.0 之前完成一轮安全与稳定性加固。

## 更新亮点

### 分享与访问密钥

- **发布链接支持访问密钥**：文档与专栏公开后可设置访问密钥，访客需输入密钥（或使用附带密钥的链接）才能查看，类似飞书的分享密码体验。
- **创建即发布**：创建文档 / 专栏时即可勾选「创建后立即公开」，并可同时设置可选的访问密钥。
- **复制链接可选是否附带密钥**：分享对话框新增「链接附带密钥」开关——附带密钥的链接访客免输入直接解锁；不附带的链接则会进入密钥输入页。
- 分享对话框、SEO 访问门（access gate）与带密钥链接的整体链路打通。

### 安全加固

- **密码哈希升级为 scrypt**：替换原先的单轮 SHA-256，采用内存硬化 KDF（参数自描述，未来可平滑升级）；存量用户登录成功后自动透明迁移，无需任何操作。
- **修复专栏挂载越权**：创建 / 编辑文档时挂载到专栏，现在会校验你对目标专栏的写权限。
- **修复错误响应的 CORS 头**：异常响应不再硬编码 `Access-Control-Allow-Origin: *`，凭据模式下前端可以正确读取错误信息。

### 稳定性

- **后台任务入队顺序统一**：摘要 / 向量化 / 转写 / 构图 / 播客任务统一为「先落库、再入队」，消除 worker 读到旧状态以及幽灵任务的竞态；入队失败会正确标记失败而不是永远卡在排队中。
- **修复 worker 数据库连接跨事件循环复用**：彻底解决任务偶发的连接挂死 / "attached to a different loop" 报错。
- **修复有发布历史的专栏取消发布时崩溃**的问题。
- api 与 celery-worker 的镜像代码（枚举 / 通知模板 / Apple 推送工具）完成一轮同步，消除漂移。

### Bug 修复与打磨

- 一批 UI 细节优化，分享对话框信息布局精简。
- 清理无用代码。
